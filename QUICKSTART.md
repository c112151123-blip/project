# 快速開始指南

## 環境準備

### Windows 系統
1. **打開命令提示符或 PowerShell**
   
2. **創建虛擬環境**
   ```cmd
   python -m venv .venv
   ```

3. **激活虛擬環境**
   ```cmd
   .venv\Scripts\activate
   ```

4. **安裝依賴包**
   ```cmd
   pip install -r requirements.txt
   ```

5. **啟動應用**
   ```cmd
   python run.py
   ```

6. **打開瀏覽器訪問**
   - 地址：http://localhost:5000
   - 首次訪問自動創建數據庫

---

### macOS / Linux 系統
1. **打開終端**

2. **創建虛擬環境**
   ```bash
   python3 -m venv .venv
   ```

3. **激活虛擬環境**
   ```bash
   source .venv/bin/activate
   ```

4. **安裝依賴包**
   ```bash
   pip install -r requirements.txt
   ```

5. **啟動應用**
   ```bash
   python run.py
   ```

6. **打開瀏覽器訪問**
   - 地址：http://localhost:5000

## 首次使用

### 角色說明
系統分為三種用戶角色：

| 角色 | 功能 |
|------|------|
| **學生** | 可報告失物、搜尋拾物、提交認領申請 |
| **教職員** | 可報告失物、搜尋拾物、提交認領申請 |
| **管理員** | 上傳拾物、管理系統內容、批准認領申請 |

### 步驟 1：創建普通用戶賬戶（學生或教職員）
1. 點擊"註冊"按鈕
2. 填寫表單：
   - 學號/編號：`STU20260001`（學生）或 `STAFF2026001`（教職員）
   - 電郵：`user@school.com`
   - 真實姓名：`測試用戶`
   - 身份：選擇「學生」或「教職員」
   - 密碼：`123456`
3. 點擊"立即註冊"

### 步驟 2：使用普通賬戶登入
1. 點擊首頁上的"登入"按鈕
2. 輸入學號/編號和密碼
3. 點擊登入

### 步驟 3：報告失物（普通用戶）
1. 登入後，在導航菜單中點擊"報告失物"
2. 填寫表單：
   - 物品名稱：`黑色iPhone 13`
   - 類別：`電話`
   - 丟失地點：`圖書館三樓`
   - 丟失日期：選擇日期
   - 詳細描述：`黑色 iPhone 13，屏幕有貼膜`
   - 聯絡電話：`13800138000`
3. 點擊"提交報告"

### 步驟 4：搜尋拾物
1. 點擊首頁的"拾物查詢"
2. 瀏覽或搜尋相符的拾物
3. 如果發現可能是自己的物品，點擊"查看詳情"
4. 提交認領申請

### 步驟 5：創建管理員賬戶（系統管理員進行）
1. 點擊"註冊"按鈕創建新賬戶
2. 填寫信息（需要設置為管理員）

> 見「進階功能」→「創建管理員賬戶」部分說明如何設置管理員

### 步驟 6：管理員登入
1. 點擊首頁的"管理員登入"（紅色按鈕）
2. 輸入管理員編號和密碼
3. 點擊"管理員登入"

### 步驟 7：管理員上傳拾物
1. 使用管理員賬戶登入
2. 在導航菜單中點擊"上傳拾物"
3. 填寫物品信息
4. 點擊"提交報告"

## 數據庫位置

首次運行後，數據庫文件會自動生成在：
```
project/lost_found.db
```

**清除所有數據**（重新開始）：
1. 停止應用（按 Ctrl+C）
2. 刪除 `lost_found.db` 文件
3. 重新啟動應用

## 常見問題解決

### 問題 1：找不到 Flask 模塊
**解決方案**：確保虛擬環境已激活
```bash
# 檢查是否激活（命令提示符前應該有 (.venv)）
pip list
```

### 問題 2：端口 5000 已被使用
**解決方案**：修改啟動方式
```bash
python run.py  # 會自動嘗試其他端口
```

### 問題 3：中文顯示亂碼
**解決方案**：確保使用支持 UTF-8 的終端

## 進階功能

### 了解系統架構
- **普通用戶**（學生、教職員）：
  - 可以報告自己丟失的物品
  - 可以搜尋系統中的拾物
  - 可以提交認領申請
  - 可以編輯和刪除自己報告的失物
  
- **管理員**：
  - 上傳系統內找到的拾物
  - 管理所有拾物記錄
  - 使用專門的管理員登入入口

### 創建管理員賬戶

**重要安全提醒：** 
- 管理員有更高的系統權限，應該謹慎分配
- 只有系統真正的管理人員才應該被設置為管理員
- 所有管理員操作都應該被監控

**方式 1：通過代碼（開發時）**
1. 打開 `app/routes.py` 文件
2. 找到 `register()` 函數，在創建 User 對象後添加：
   ```python
   # 開發環境：設置為管理員
   if id_number == 'ADMIN_TEST':  # 只為特定編號設置
       user.role = 'admin'
   ```

**方式 2：直接數據庫修改（生產環境）**
1. 停止應用
2. 使用 SQLite 工具（如 DB Browser for SQLite）打開 `instance/lost_found.db`
3. 在 `user` 表中，找到要提升為管理員的用戶
4. 將該用戶的 `role` 字段改為 `'admin'`
5. 保存數據庫
6. 重啟應用
7. 該用戶現在可以使用"管理員登入"入口

**方式 3：通過管理代碼（推薦）**
> 創建一個只有系統管理員知道的管理端點，用於提升賬戶

### 安全最佳實踐
1. **強密碼**：確保管理員賬戶使用強密碼（至少8位，包含大小寫和數字）
2. **定期檢查**：定期檢查誰被設置為管理員
3. **監控日誌**：考慮添加操作日誌來追蹤管理員操作
4. **最小權限原則**：只給必要的人員管理員權限

### 修改物品分類
編輯 `app/routes.py`，在各路由函數中搜索 `categories` 列表並修改：
```python
categories = ['電話', '錢包', '鑰匙', '衣服', '眼鏡', '書包', '手錶', '其他']
```

### 自定義樣式
編輯 `app/static/css/style.css` 修改應用外觀。

### 啟用 HTTPS
在生產環境中，一定要：
1. 使用 HTTPS 加密所有通訊
2. 設置 SESSION_COOKIE_SECURE = True
3. 設置 SESSION_COOKIE_HTTPONLY = True

## 部署到生產環境

當準備部署到生產環境時：

1. **安裝生產服務器**
   ```bash
   pip install gunicorn
   ```

2. **更新配置**
   - 修改 `.env` 中的 `FLASK_ENV=production`
   - 生成強的 `SECRET_KEY`

3. **運行應用**
   ```bash
   gunicorn -w 4 -b 0.0.0.0:5000 run:app
   ```
