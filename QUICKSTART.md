# 快速開始指南

## 環境準備

### Windows 系統
1. **打開命令提示符或 PowerShell**
   
2. **創建虛擬環境**
   ```cmd
   python -m venv .venv
   ```

3. **激活虛擬環境**
   ```cmd
   .venv\Scripts\activate
   ```

4. **安裝依賴包**
   ```cmd
   pip install -r requirements.txt
   ```

5. **啟動應用**
   ```cmd
   python run.py
   ```

6. **打開瀏覽器訪問**
   - 地址：http://localhost:5000
   - 首次訪問自動創建數據庫

---

### macOS / Linux 系統
1. **打開終端**

2. **創建虛擬環境**
   ```bash
   python3 -m venv .venv
   ```

3. **激活虛擬環境**
   ```bash
   source .venv/bin/activate
   ```

4. **安裝依賴包**
   ```bash
   pip install -r requirements.txt
   ```

5. **啟動應用**
   ```bash
   python run.py
   ```

6. **打開瀏覽器訪問**
   - 地址：http://localhost:5000

## 首次使用

### 步驟 1：創建測試賬戶（管理員）
1. 點擊"註冊"按鈕
2. 填寫表單：
   - 學號/編號：`ADMIN001`（用於登錄的標識）
   - 電郵：`admin@school.com`
   - 真實姓名：`管理員`
   - 身份：選擇「學生」或「教職員」
   - 密碼：`123456`
3. 點擊"立即註冊"

> **重要**：只有管理員可以上傳失物和拾物。要創建管理員賬戶，需要手動編輯代碼。請參考「進階功能」部分。

### 步驟 2：創建普通學生或教職員賬戶（可選）
1. 點擊"註冊"按鈕
2. 填寫表單：
   - 學號/編號：`STU20260001`（學生）或 `STAFF2026001`（教職員）
   - 電郵：`user@school.com`
   - 真實姓名：`普通用戶`
   - 身份：選擇相應身份
   - 密碼：`123456`
3. 點擊"立即註冊"

### 步驟 3：登入系統
1. 在登入頁面輸入學號/編號（例如 `STU20260001`）
2. 輸入密碼
3. 點擊登入

### 步驟 4：管理員上傳失物
1. 使用管理員賬戶登入
2. 在導航菜單中點擊"上傳失物"
3. 填寫表單：
   - 物品名稱：`黑色iPhone 13`
   - 類別：`電話`
   - 丟失地點：`圖書館三樓`
   - 丟失日期：選擇今天或昨天
   - 詳細描述：`黑色 iPhone 13，屏幕有貼膜，背面有一道小劃痕`
   - 聯絡電話：`13800138000`
4. 點擊"提交報告"

### 步驟 5：管理員上傳拾物
1. 在導航菜單中點擊"上傳拾物"
2. 填寫與失物相似的物品信息
3. 點擊"提交報告"

### 步驟 6：普通用戶搜尋和認領
1. 使用普通學生或教職員賬戶登入
2. 點擊"失物尋找"或"拾物查詢"瀏覽物品
3. 找到相符的物品後，提交認領申請
4. 等待管理員或其他用戶批准

## 數據庫位置

首次運行後，數據庫文件會自動生成在：
```
project/lost_found.db
```

**清除所有數據**（重新開始）：
1. 停止應用（按 Ctrl+C）
2. 刪除 `lost_found.db` 文件
3. 重新啟動應用

## 常見問題解決

### 問題 1：找不到 Flask 模塊
**解決方案**：確保虛擬環境已激活
```bash
# 檢查是否激活（命令提示符前應該有 (.venv)）
pip list
```

### 問題 2：端口 5000 已被使用
**解決方案**：修改啟動方式
```bash
python run.py  # 會自動嘗試其他端口
```

### 問題 3：中文顯示亂碼
**解決方案**：確保使用支持 UTF-8 的終端

## 進階功能

### 了解系統架構
- **一般用戶**（學生、教職員）：可以搜尋失物和拾物，提交認領申請
- **管理員**：負責上傳和管理失物及拾物記錄
- 用戶通過**學號或編號**（ID）進行登錄和識別，而非用戶名
- 真實姓名在儀表板中顯示

### 創建管理員賬戶
有兩種方式：

**方式 1：通過代碼（開發用）**
1. 打開 `app/routes.py` 文件
2. 找到 `register()` 函數
3. 在創建 User 對象後添加：
   ```python
   user.role = 'admin'  # 設置為管理員
   ```

**方式 2：直接數據庫修改（快速）**
1. 停止應用
2. 使用 SQLite 工具（如 DB Browser for SQLite）打開 `instance/lost_found.db`
3. 在 `user` 表中，將所需用戶的 `role` 字段改為 `'admin'`
4. 保存並重啟應用

### 修改物品分類
編輯 `app/routes.py`，在各路由函數中搜索 `categories` 列表並修改：
```python
categories = ['電話', '錢包', '鑰匙', '衣服', '眼鏡', '書包', '其他']
```

### 自定義樣式
編輯 `app/static/css/style.css` 修改應用外觀。

## 部署到生產環境

當準備部署到生產環境時：

1. **安裝生產服務器**
   ```bash
   pip install gunicorn
   ```

2. **更新配置**
   - 修改 `.env` 中的 `FLASK_ENV=production`
   - 生成強的 `SECRET_KEY`

3. **運行應用**
   ```bash
   gunicorn -w 4 -b 0.0.0.0:5000 run:app
   ```
